# 项目规则

**版本**: 1.0 | **日期**: [YYYY-MM-DD]

---

## 核心原则

1. **简洁优先 (KISS)**: 选择最简单的方案
2. **不重复 (DRY)**: 相同逻辑只实现一次
3. **不过度 (YAGNI)**: 只实现当前需要的

---

## 架构约束

| 约束 | 规则 |
|------|------|
| 项目数 | ≤ 3 个 |
| 文件行数 | ≤ 500 行 |
| 函数长度 | ≤ 50 行 |

---

## 命名约定

| 类型 | 规则 | 示例 |
|------|------|------|
| 文件 | 小写下划线/kebab | `user_service.py` |
| 类 | 大驼峰 | `UserService` |
| 函数/变量 | 小写下划线 | `get_user()` |
| 常量 | 大写下划线 | `MAX_COUNT` |

---

## Git 规范

```
<类型>(<范围>): <描述>

类型: feat|fix|docs|style|refactor|test|chore
示例: feat(user): 添加用户注册功能
```
