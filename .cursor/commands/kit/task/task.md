# 任务清单: [功能名称]

## 用户输入

```text
$ARGUMENTS
```

---

## 核心目的

按需求实现代码更新，分步骤执行。

---

## 执行规范

**必须遵循** `.cursor/commands/kit/task/rules.md` 中的规范：
- 如果是前端代码，遵循 `.cursor/rules/` 目录下的规范文件

---

## 执行步骤

### 1. 获取任务列表

从上一步的计划中获取步骤规划，按顺序执行。

### 2. 逐步执行

对于每个步骤：

```markdown
### 执行 Step [N]: [步骤名称]

🔄 状态：进行中

**操作内容：**
[具体的代码修改]

✅ 完成
```

### 3. 执行进度追踪

```markdown
## 执行进度

- [x] Step 1: [名称] ✅
- [x] Step 2: [名称] ✅
- [ ] Step 3: [名称] 🔄
- [ ] Step 4: [名称] ⏳
```

---

## 完成条件

所有步骤执行完成后，输出：

```
✅ 任务执行完成

📊 执行统计：
- 完成步骤：[N]/[Total]
- 修改文件：[文件列表]
- 新增文件：[如有]
- 删除文件：[如有]
```

---

## 🔄 确认流转

**任务完成后，需要用户确认才能进入检查阶段。**

输出确认提示：

```
🔄 是否进入 [检查阶段] 进行代码检查？

回复 Y 继续，或提出需要修改的地方
```

等待用户回复：
- 用户回复 `Y` 或 `是` → 自动执行 `.cursor/commands/kit/check/check.md`
- 用户有其他意见 → 根据反馈继续修改代码
