# 技术研究: [研究主题]

## 用户输入

```text
$ARGUMENTS
```

---

## 核心目的

针对需求中的技术问题进行深入研究，输出可行的技术方案。

---

## 执行步骤

### 1. 理解问题

a. 根据用户提问，明确研究目标
b. 识别关键技术点和约束条件

### 2. 分析项目代码

a. 分析当前代码库结构
b. 识别现有实现模式
c. 找出可复用的组件/模块

### 3. 搜索相关文档

a. 查阅官方文档
b. 搜索最佳实践
c. 参考类似项目实现

### 4. 方案对比

结合代码和文档，给出不同的技术方案：

```markdown
## 方案对比

| 方案 | 优点 | 缺点 | 改动量 | 风险 |
|------|------|------|--------|------|
| 方案 A | ... | ... | 小/中/大 | 低/中/高 |
| 方案 B | ... | ... | 小/中/大 | 低/中/高 |
| 方案 C | ... | ... | 小/中/大 | 低/中/高 |
```

### 5. 推荐方案

**选择标准**（按优先级排序）：
1. 改动少
2. 副作用小
3. 稳定性好
4. 可维护性强

---

## 完成条件

输出研究结论：

```
✅ 技术研究完成

🎯 推荐方案：[方案名称]
📝 理由：
  - [理由1]
  - [理由2]
  - [理由3]

⚠️ 注意事项：[实施时需要注意的点]
```

---

## 📌 说明

此命令为 **辅助命令**，可在以下场景调用：
- plan 阶段遇到技术选型问题
- 需要对比多种实现方案
- 需要深入研究某个技术点

调用方式：`/k/research [研究问题]`
